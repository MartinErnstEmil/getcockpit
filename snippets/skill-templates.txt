# <Skill Title>

Triggered when the user asks for a code review on <subject>.

## How to apply

1. Identify scope (one commit / one PR / one file).
2. Spawn three reviewer agents in parallel (Reuse / Quality / Efficiency).
3. Wait for all three; consolidate findings into a single ranked list.
4. Implement HIGH+CRITICAL fixes; defer MEDIUM unless asked.
5. Commit fixes as one "simplify" commit.

## Don'ts

- Don't run reviewers sequentially — that's 3× the wall time.
- Don't apply LOW findings without asking; they're often taste calls.
